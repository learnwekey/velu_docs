title: Docs site structure
description: Learn how a Velu docs site organizes pages, configuration, assets, and API specifications.
---

A Velu docs site is a directory containing `velu.json`, Markdown or MDX pages,
and any assets or API specifications used by the site. In a docs-as-code
workflow, Git tracks these files and Velu uses them to render the
documentation.

  <File name="favicon.svg" />

## Core files

<Field name="velu.json" type="JSON" required>
Controls the project name, branding, navigation, API documentation, contextual
actions, and other site-wide behavior.
</Field>

<Field name="*.md" type="Markdown" required>
Contains page headings, prose, code examples, images, and links. Use MDX when a
page needs built-in Velu components.
</Field>

<Field name="openapi.json" type="OpenAPI">
Defines an API reference when the project uses generated API documentation.
OpenAPI specifications can also use YAML or a remote URL.
</Field>

<Field name="images/" type="directory">
Stores screenshots, diagrams, and other project assets. You may choose another
directory name as long as page references match it.
</Field>

## Page titles

Start a Markdown page with a descriptive level-one heading:

```md
# Configure authentication
```

The heading identifies the page to readers, search, and AI tools. Keep it
specific to the task or subject covered by the page.

## Map files to URLs

Page paths in `velu.json` are relative to the project root and omit the `.md`
extension:

```json
{
  "group": "Guides",
  "pages": ["quickstart", "guides/configuration", "guides/deployment"]
}
```

This configuration maps:

| File | URL |
| --- | --- |
| `quickstart.md` | `/quickstart` |
| `guides/configuration.md` | `/guides/configuration` |
| `guides/deployment.md` | `/guides/deployment` |

<Callout type="warning">

Add a page to `navigation` before expecting it to render or be validated. A
misspelled path or missing file causes `velu lint` to report an error.

</Callout>

## Use built-in components

Velu components are available directly in MDX. Do not import them:

```mdx
<Callout type="tip">
  Preview the page before publishing it.
</Callout>
```

Component names and properties are case-sensitive. Run `velu lint` after
adding or changing component markup.

## Create a new local project

Use `velu init` when you need a standalone starter project:

```bash
mkdir your-docs-repo
cd your-docs-repo
velu init
velu run
```

The command scaffolds the current directory and refuses to replace an existing
Velu configuration.

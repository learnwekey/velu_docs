---
title: Writing API
description: Create, update, and publish Velu documentation pages from your app.
---

Use the Velu Writing API to generate docs pages from product workflows,
release pipelines, or internal tools. A writing request creates an MDX file,
adds metadata, and can place the page in navigation in one call.

<ApiPath method="post" path="/v1/writing/pages" />

<Callout type="tip">
  Use the Writing API for trusted automation such as release notes,
  generated integration guides, and API migration pages. Keep core docs in
  your repo when you need normal review, preview, and rollback workflows.
</Callout>

## Authentication

Send a project API key with `docs:write` access in the `Authorization`
header.

```bash
Authorization: Bearer velu_live_xxxxxxxxx
Content-Type: application/json
Idempotency-Key: release-2026-06-25
```

## Create a Page

Creates a new page or updates an existing page when `path` already exists.
Paths are relative to the docs root and should not include `.mdx`.

<Field name="path" type="string" required>
  Relative page path, for example `guides/release-notes` or
  `api-reference/writing-api`.
</Field>

<Field name="title" type="string" required>
  Page title used in frontmatter, browser metadata, and navigation labels.
</Field>

<Field name="description" type="string">
  Short summary used for search, cards, and page metadata.
</Field>

<Field name="content_mdx" type="string" required>
  MDX body for the page. It can include Markdown, fenced code blocks, and
  Velu components such as `Callout`, `CodeGroup`, `Steps`, and `Field`.
</Field>

<Field name="navigation" type="object">
  Optional placement instructions for adding the page to an existing tab and
  group.
</Field>

<Field name="publish_mode" type="enum" default="draft">
  Use `draft` to stage the page, `preview` to create a shareable preview, or
  `live` to publish immediately.
</Field>

### Example Request

```bash
curl https://api.veludocs.com/v1/writing/pages \
  -H "Authorization: Bearer $VELU_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: release-2026-06-25" \
  -d '{
    "path": "guides/summer-release",
    "title": "Summer Release Guide",
    "description": "New features and migration notes for the summer release.",
    "publish_mode": "preview",
    "navigation": {
      "tab": "Guides",
      "group": "Release Notes",
      "position": "top"
    },
    "content_mdx": "## Overview\n\n<Callout type=\"note\">This page was generated from the release pipeline.</Callout>\n\n```ts\nawait velu.pages.create({ title: \"Summer Release Guide\" });\n```"
  }'
```

### Example Response

```json
{
  "id": "page_2Y7bQmWq9h",
  "path": "guides/summer-release",
  "status": "preview",
  "preview_url": "https://preview.veludocs.com/acme/guides/summer-release",
  "updated_at": "2026-06-25T12:15:30Z"
}
```

## Component Types

Use components when generated content needs structure beyond plain Markdown.

<Field name="callout" type="component">
  Highlights important context with `note`, `tip`, `warning`, `danger`,
  `info`, or `check` styling.
</Field>

<Field name="code_snippet" type="component">
  Renders a fenced code block or `CodeGroup` for language-specific examples.
</Field>

<Field name="steps" type="component">
  Breaks a workflow into ordered `Step` items for setup, migration, or release
  instructions.
</Field>

<Field name="field" type="component">
  Documents request fields, configuration options, or response properties with
  type and required metadata.
</Field>

### Generated MDX Snippet

````mdx
<Callout type="note">
  This section was generated by the Velu Writing API.
</Callout>

```ts
const page = await velu.pages.create({
  path: "guides/summer-release",
  title: "Summer Release Guide",
});
```
````

<Callout type="warning">
  Validate generated MDX before publishing live. Invalid component props or
  unclosed tags will block the page from rendering.
</Callout>

## Update a Page

<ApiPath method="patch" path="/v1/writing/pages/{page_id}" />

Use updates when you already know the page ID. The request accepts any of the
create fields except `path`, unless `allow_path_rename` is set to `true`.

```json
{
  "title": "Summer Release Guide",
  "description": "Updated migration details for the summer release.",
  "publish_mode": "live"
}
```

## Publishing Flow

<Steps>
  <Step title="Generate">
    Build the MDX body from your release notes, schema changes, or internal
    source of truth.
  </Step>
  <Step title="Preview">
    Create the page with `publish_mode: "preview"` and check the returned
    `preview_url`.
  </Step>
  <Step title="Publish">
    Promote the page with `publish_mode: "live"` after validation passes.
  </Step>
</Steps>

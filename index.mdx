Velu is an AI-native documentation platform for creating, previewing, and
publishing documentation. Write and publish in the dashboard, or connect a Git
repository and work locally with Markdown, AI coding agents, and Velu's command
line tools.

<Callout type="note">

You do not need to connect a Git repository to access the dashboard or use the
web editor. Connect a documentation repository when you want a docs-as-code
workflow, Git-based automation, or self-updating documentation.

</Callout>

## Choose how you write

Velu supports two authoring workflows:

<Columns cols={2}>

<Card title="Docs as code" icon="git-branch" cta={{ label: "Use docs as code", href: "/quickstart#use-docs-as-code" }}>
Work in your local repository with Codex, Claude Code, or your editor. Preview,
validate, and review changes before pushing them. This workflow requires a
connected Git repository.
</Card>

<Card title="Dashboard editor" icon="panels-top-left" cta={{ label: "See the editor workflow", href: "/quickstart#use-the-dashboard-editor" }}>
Write and publish from the browser without connecting a Git repository or
installing local tools.
</Card>

</Columns>

Docs as code is the recommended workflow because it gives your team Git
history, pull-request review, local previews, and access to AI coding agents
with repository context.

## How a Velu project fits together

<CardGroup>

<Card title="Content" icon="files">
Markdown pages contain your documentation, code examples, images, and built-in
Velu components.
</Card>

<Card title="Configuration" icon="settings">
`velu.json` controls navigation, branding, API documentation, AI actions, and
site-wide behavior.
</Card>

<Card title="Delivery" icon="cloud-upload">
Publish directly from the dashboard, deploy changes from a connected Git
repository, or build a static site locally.
</Card>

</CardGroup>

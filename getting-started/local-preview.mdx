---
title: Preview locally
description: Start the Velu development server and review documentation changes as you write.
---

`velu dev` starts a live preview and watches the project for changes.

## Start the preview

From the directory containing `velu.json`, run:

```bash
velu dev
```

You can also pass a project directory:

```bash
velu dev path/to/yourcompany-docs
```

Velu prints the preview URL when the server is ready. The default is:

```text
http://localhost:8358
```

## Review changes as you write

The preview reloads when you change an `.mdx` page or `velu.json`. Use it to
review navigation, built-in components, code examples, images, branding, and
generated API pages before publishing.

<Callout type="note">

Search and hosted AI services are unavailable or use preview behavior under
`velu dev`. Validate their production behavior on a published preview or site.

</Callout>

## Use another port

Set the `PORT` environment variable for the command:

```bash
PORT=5180 velu dev
```

Velu then serves the site at `http://localhost:5180`.

## Stop the preview

Return to the terminal running Velu and press `Ctrl+C`.

## Troubleshoot the preview

<AccordionGroup>

<Accordion title="Velu cannot find velu.json">

Run the command from the project root or pass the project directory explicitly.
Use `velu init <project-name>` only when creating a separate new project.

</Accordion>

<Accordion title="Port 8358 is already in use">

Start the preview on another port:

```bash
PORT=5180 velu dev
```

</Accordion>

<Accordion title="A page does not appear">

Confirm that the `.mdx` file exists and that its extensionless path appears in
`navigation` in `velu.json`. Then run `velu validate`.

</Accordion>

<Accordion title="Changes do not reload">

Check the terminal for an MDX or configuration error. Correct it, save the
file, and restart `velu dev` if the process stopped.

</Accordion>

</AccordionGroup>

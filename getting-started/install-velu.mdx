title: Install Velu

Install `@veluai/velu` to scaffold, preview, and validate a documentation
project from your terminal.

## Requirements

- Node.js 18 or later.
- npm, or another package runner that can execute npm packages.
- Git for the recommended docs-as-code workflow.

Check the installed Node.js version:

```bash
node --version
```

## Install the CLI

<CodeGroup>
  <CodeBlock title="Install globally" language="bash">
    {`npm install --global @veluai/velu`}
  </CodeBlock>
  <CodeBlock title="Run with npx" language="bash">
    {`npx @veluai/velu dev .`}
  </CodeBlock>
</CodeGroup>

A global installation provides the shorter `velu` command. Use npx when you
do not want to install the package globally.

Verify a global installation:

```bash
velu --version
```

## Available commands

| Command | Purpose |
| --- | --- |
| `velu init <project-name>` | Create a new project directory from the starter template. |
| `velu dev [directory]` | Start the live preview for a project. The directory defaults to the current directory. |
| `velu validate [directory]` | Check configuration, navigation, assets, MDX, components, and literal constrained properties. |
| `velu update` | Install the latest `@veluai/velu` release globally. |

The public CLI does not expose a standalone production-build command. Hosted
projects are built during Velu's publishing workflow.

`velu dev` checks for a newer published CLI before booting. Set
`VELU_NO_UPDATE=1` to skip that auto-update gate for a session.

## Initialize a project

Run `init` from the directory that should contain the new project directory:

```bash
velu init yourcompany-docs
cd yourcompany-docs
velu dev
```

The target directory must be absent or empty. Velu creates `velu.json`, a
favicon, an example OpenAPI document, and starter `.mdx` pages.

<Callout type="warning">

Do not run `velu init .` in a non-empty repository. The command refuses to
replace files in a non-empty target directory.

</Callout>

## Troubleshoot installation

<AccordionGroup>

<Accordion title="The velu command is not found">

Open a new terminal after installing globally. If the command remains
unavailable, run the package through npx:

```bash
npx @veluai/velu dev .
```

</Accordion>

<Accordion title="The global installation is denied">

Use npx instead of changing system-directory permissions:

```bash
npx @veluai/velu validate .
```

</Accordion>

<Accordion title="The Node.js version is unsupported">

Install Node.js 18 or later, reopen the terminal, and run `node --version`
before retrying.

</Accordion>

<Accordion title="Velu cannot find the project">

Pass the project directory to the command, or run it from the directory that
contains `velu.json`.

</Accordion>

</AccordionGroup>
